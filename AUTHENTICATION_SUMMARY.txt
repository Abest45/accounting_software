📊 API ENDPOINTS (12 Total)
═══════════════════════════════════════════════════════════════════════════════

USER AUTHENTICATION (3 Endpoints)
──────────────────────────────────
POST   /api/auth/register
       Create new user account
       Input: username, email, password, firstName, lastName
       Returns: userId, email, token

POST   /api/auth/login
       User login with credentials
       Input: email, password, rememberMe
       Returns: token, user data

PUT    /api/auth/profile
       Update user profile (authenticated)
       Input: firstName, lastName, email, newPassword
       Returns: updated user data


ADMIN AUTHENTICATION (2 Endpoints)
───────────────────────────────────
POST   /api/auth/admin/register
       Request admin account (requires approval)
       Input: firstName, lastName, email, phone, company, password
       Returns: registration request acknowledgment

POST   /api/auth/admin/login
       Admin login with optional 2FA
       Input: email, password, twoFactorCode
       Returns: token, admin user data


PASSWORD RECOVERY (3 Endpoints)
────────────────────────────────
POST   /api/auth/forgot-password
       Initiate password recovery
       Input: email
       Returns: recovery initiated message (no user reveal)

POST   /api/auth/validate-reset-token
       Verify reset token validity
       Input: token
       Returns: token valid status, email, firstName

POST   /api/auth/reset-password
       Complete password reset
       Input: token, password, confirmPassword
       Returns: reset successful message

🐛 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Problem: Emails not sending
Solution: 
  1. Check .env SMTP configuration
  2. Verify email credentials
  3. Check firewall/port 587 access
  4. Enable "Less secure apps" (Gmail)
  5. Check spam folder
  6. Review logs for errors

Problem: "Token is invalid or expired"
Solution:
  1. Token expires after 30 minutes
  2. Request a new recovery link
  3. Verify URL is complete
  4. Check system time on server

Problem: "Account is locked"
Solution:
  1. Wait 30 minutes for auto-unlock
  2. Contact admin for manual unlock
  3. Use password recovery
  4. Try different browser/device

Problem: CORS errors
Solution:
  1. Check CORS_ORIGIN in .env
  2. Restart backend server
  3. Clear browser cache/cookies
  4. Check request headers

Problem: Form validation fails
Solution:
  1. Check password requirements (8+ chars)
  2. Verify email format (user@domain.com)
  3. Check phone number (10+ digits)
  4. Review error messages in console

Problem: 2FA code doesn't work
Solution:
  1. Check server time matches client
  2. Code valid for 30 seconds only
  3. Verify TOTP library installed
  4. Check timezone settings
