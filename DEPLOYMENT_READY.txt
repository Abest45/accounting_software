🧪 TESTING THE API:

1. REGISTER USER:
   curl -X POST http://localhost:5000/api/auth/register \
     -H "Content-Type: application/json" \
     -d '{
       "username":"testuser",
       "email":"test@example.com",
       "password":"SecurePass@123"
     }'

2. LOGIN:
   curl -X POST http://localhost:5000/api/auth/login \
     -H "Content-Type: application/json" \
     -d '{
       "email":"test@example.com",
       "password":"SecurePass@123"
     }'
   
   Copy the returned token!

3. GET DASHBOARD STATS:
   curl -X GET http://localhost:5000/api/dashboard/stats \
     -H "Authorization: Bearer YOUR_TOKEN"

4. CREATE INVOICE:
   curl -X POST http://localhost:5000/api/invoices \
     -H "Authorization: Bearer YOUR_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{
       "clientName":"Acme Corp",
       "clientEmail":"contact@acme.com",
       "invoiceDate":"2024-12-01",
       "dueDate":"2024-12-31",
       "items":[{"description":"Service","quantity":1,"price":5000}],
       "taxRate":10
     }'

🆘 TROUBLESHOOTING:

PROBLEM: "Database connection refused"
SOLUTION: 
  - Check PostgreSQL is running
  - Verify credentials in .env
  - Create database: createdb finanalytics_db

PROBLEM: "Port 5000 already in use"
SOLUTION:
  - Change PORT in .env
  - Or kill process: lsof -i :5000; kill -9 PID

PROBLEM: "CORS errors in frontend"
SOLUTION:
  - Update CORS_ORIGIN in .env
  - Restart server: npm run dev

PROBLEM: "Authentication token expired"
SOLUTION:
  - Login again (tokens expire after 7 days)
  - Check Authorization header format: "Bearer TOKEN"

For more help, see: backend/QUICKSTART.md
